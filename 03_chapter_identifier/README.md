# Chapter 03 - Identifiers, Naming Rules & Comments

This chapter covers how to name variables correctly and how to document code using comments. It also includes a handy keyboard shortcuts reference.

## Files

| File | Description |
|------|-------------|
| `05_identifier-rules.js` | Basic rules for valid identifiers |
| `06_ruleforidentifier.js` | Extended identifier rules |
| `07_ruleforidentifiercases.js` | Naming conventions: `camelCase`, `snake_case`, `PascalCase` |
| `08_comment.js` | Single-line (`//`) and multi-line (`/* */`) comments |
| `antigravity_keyboard_shortcut.md` | VS Code, Browser, Windows, Mac & Terminal shortcuts |

## Key Takeaways

- Identifiers must start with a letter, underscore (`_`), or dollar sign (`$`).
- They are case-sensitive.
- Use meaningful names to improve readability.
- Comments make code easier to maintain.
